# ResearchOS

Персональная система для написания академических работ с Claude Code.
Дипломные, магистерские, курсовые, статьи, эссе — каждая работа в отдельном
изолированном подпроекте с собственным стилем, автором и форматом.

---

## Первый запуск — что сделать один раз

### 1. Установить зависимости

```bash
# Python-пакеты (для скриптов)
pip install python-docx pdfplumber
```

**Pandoc** (для экспорта в DOCX) — установить один раз на компьютер:

| ОС | Команда |
|----|---------|
| macOS | `brew install pandoc` |
| Windows | `winget install JohnMacFarlane.Pandoc` |
| Linux | `sudo apt-get install pandoc` |
| Любая | Скачать установщик: https://pandoc.org/installing.html |

### 2. Заполнить Me.md

Открой `Me.md` в корне проекта и заполни все разделы — это главный файл о тебе.
Claude читает его в начале каждой сессии и адаптирует поведение под твой уровень,
предпочтения и стиль взаимодействия.

**Разделы Me.md:**
- ФИО и специализация
- Уровень подготовки (бакалавр / магистр / аспирант / другое)
- Область знаний и дисциплины
- Опыт академического письма — что писала, что было сложно
- Любимые авторы и тексты (ориентир стиля)
- Что раздражает в академическом письме
- Как хочешь работать с Claude (насколько детально, какой тон)
- Сильные и слабые стороны

## Создание нового подпроекта

Каждая работа — отдельная папка в `projects/`. Стиль, голос, требования и источники
одного проекта **не переносятся** в другой.

### Шаг 1 — Запустить скилл

```
/new-project
```

Claude спросит название и тип работы, создаст папку `projects/<slug>/` со всей
структурой и выведет чеклист что заполнить.

### Шаг 2 — Заполнить CONTEXT.md

Самый важный файл. От него зависит всё — структура, стиль, поиск источников.

```
projects/<slug>/CONTEXT.md
```

**Что заполнить обязательно:**

| Поле | Что писать |
|------|-----------|
| Название работы | Полное официальное название |
| Тип | дипломная / магистерская / курсовая / статья / эссе |
| Дисциплина и кафедра | Точное название |
| Научный руководитель | ФИО, или "нет" |
| Дедлайн сдачи | ДД.ММ.ГГГГ |
| Объём | Страниц / слов, что включается/не включается |
| Стандарт оформления | ГОСТ / APA / MLA / требования кафедры |
| Язык написания | ru / en / другой |
| **Исследовательский вопрос** | Чёткий вопрос, на который отвечает работа |
| Гипотеза | Если есть — формулируй конкретно |
| Цель и задачи | Цель одна, задачи — шаги к ней |
| Объект и предмет | Что изучается и с какой стороны |
| Методология | Теоретические и эмпирические методы |
| Что уже есть | Черновики, собранные источники, данные |
| Ограничения | Требования руководителя, что нельзя |

> Исследовательский вопрос — самое важное. Всё остальное строится вокруг него.
> Если вопрос размытый — попроси Claude помочь его сформулировать до начала работы.

### Шаг 3 — Заполнить FORMAT.md

```
projects/<slug>/FORMAT.md
```

Настройки форматирования для экспорта в DOCX. По умолчанию стоят настройки ГОСТ.

**Что проверить / изменить:**

| Параметр | По умолчанию | Варианты |
|----------|-------------|---------|
| `language` | `ru` | `en`, `de`, `fr` |
| `font_family` | `Times New Roman` | `Arial`, `Calibri` |
| `font_size` | `14` | `12`, `14` |
| `line_spacing` | `1.5` | `1.0`, `2.0` |
| `margin_left` | `30` (мм) | по требованию кафедры |
| `margin_right` | `15` (мм) | по требованию кафедры |
| `citation_style` | `GOST` | `APA`, `MLA`, `Chicago` |
| `citation_format` | `footnote` | `inline`, `endnote` |

Если кафедра не предъявляет особых требований — FORMAT.md можно не трогать.

### Шаг 4 — Заполнить AUTHOR.md

```
projects/<slug>/AUTHOR.md
```

Описывает голос и стиль письма для этой конкретной работы. Чем точнее —
тем лучше Claude воспроизводит нужный стиль при написании черновиков.

**Что заполнить:**
- Общая характеристика стиля (2-5 предложений своими словами)
- Уровень сложности текста
- Длина предложений — короткие / средние / длинные / смешанные
- Использование пассивного залога — часто / умеренно / редко
- Использование первого лица — "я считаю" / "автор полагает" / запрещено
- Типичные конструкции и любимые обороты
- Чего автор избегает
- **2-3 реальных фрагмента твоих текстов** (100-300 слов) — самое важное

> Фрагменты твоих текстов — главный ориентир. Вставь абзацы из прошлых работ,
> которые тебе нравятся. Это точнее любого описания.

### Шаг 5 — Заполнить STYLE.md

```
projects/<slug>/STYLE.md
```

Правила письма для этой работы. Claude проверяет каждый генерируемый текст
по этому файлу.

**Что заполнить:**
- Тон — строго академический / с авторским присутствием / полуформальный
- Запрещённые слова (конкретный список: "осуществлять", "является" и т.д.)
- Запрещённые обороты ("В данной работе рассматривается...", "Следует отметить...")
- Допустимые академические клише
- Формат ввода цитат
- 1-2 образцовых абзаца как эталон стиля

## Рабочий процесс

### Построить структуру

```
/build-outline
```

Читает CONTEXT.md, генерирует 3 варианта оглавления с обоснованием.
Итерирует до финального плана, сохраняет в `outline/outline_v1.md`.

### Загрузить литературу

1. Скопируй PDF/DOCX в `projects/<slug>/sources/files/`
2. Запусти для каждого файла:
   ```
   /process-source projects/<slug>/sources/files/имя_файла.pdf
   ```
   Claude извлечёт текст, создаст карточку в `sources/notes/` и предложит BibTeX.

3. Найти направления для поиска новых источников:
   ```
   /find-sources тема или вопрос
   ```

### Писать разделы

```
/write-section Введение
/write-section 2.1 Теоретические основы
```

Черновик сохраняется в `chapters/`. В тексте ставятся флаги:
- `[VERIFY]` — факт требует проверки
- `[SOURCE NEEDED]` — нужна ссылка
- `[WEAK ARG]` — аргумент требует усиления

### Работать с источниками и цитатами

```
/find-quote гендерный контракт в советском обществе
/check-citations
```

`/find-quote` ищет цитаты из обработанных карточек notes/.
`/check-citations` строит таблицу всех незакрытых флагов с предложениями.

### Писать обзор литературы

```
/literature-review весь обзор
/literature-review теории репрезентации
```

Читает карточки из notes/, группирует источники по темам или хронологии,
пишет аналитический обзор (не реферативный список).

### Ревью и проверки

```
/review-section chapters/01_vvedenie.md   ← структура, логика, источники
/style-check chapters/02_teoria.md        ← соответствие STYLE.md
/argument-check chapters/03_analiz.md     ← карта аргументации, логические ошибки
/consistency-check                         ← связность всей работы целиком
```

### Написать аннотацию

```
/abstract-write
```

После того как основная работа написана. Создаёт краткий и развёрнутый варианты.

### Экспорт в DOCX

```
/export-thesis
/export-thesis diploma_un_legitimation_speeches
/export-thesis diploma_un_legitimation_speeches final_draft
```

Скилл автоматически:
1. Проверяет установку Pandoc — если нет, даёт инструкцию по установке (macOS / Windows / Linux)
2. Генерирует `reference.docx` из `FORMAT.md` (шрифт, поля, межстрочный)
3. Собирает все главы из `chapters/*.md` в правильном порядке (00_, 01_, 02_…)
4. Подключает `bibliography.bib` если есть
5. Запускает Pandoc с оглавлением → готовый DOCX в `output/thesis_YYYY-MM-DD.docx`

Если нужен только шаблон форматирования без полного экспорта:
```
/create-reference-doc projects/<slug>
```

## Все скиллы

### Написание и структура

| Скилл | Что делает |
|-------|-----------|
| `/new-project` | Создаёт подпроект из шаблонов, выводит чеклист заполнения |
| `/research-plan` | Детальный план работы с этапами и дедлайнами |
| `/build-outline` | 3 варианта оглавления с обоснованием, итерация, сохранение |
| `/write-section <раздел>` | Черновик раздела по STYLE.md + AUTHOR.md с флагами |
| `/abstract-write` | Аннотация: актуальность → цель → методы → результаты → выводы |

### Качество текста

| Скилл | Что делает |
|-------|-----------|
| `/review-section <файл>` | Ревью по чеклисту: логика, аргументация, источники, стиль |
| `/style-check <файл>` | Нарушения STYLE.md с таблицей правок |
| `/argument-check <файл>` | Карта аргументации, логические ошибки, контраргументы |
| `/consistency-check` | Связность всей работы: главы ↔ ИВ ↔ заключение |

### Литература и источники

| Скилл | Что делает |
|-------|-----------|
| `/find-sources <тема>` | Направления поиска, термины, базы данных по дисциплине |
| `/process-source <файл>` | PDF/DOCX → карточка в notes/ + BibTeX запись |
| `/find-quote <тема>` | Ищет цитаты и тезисы из обработанных карточек notes/ |
| `/literature-review <тема>` | Аналитический обзор литературы из notes/ |
| `/check-citations` | Таблица всех [SOURCE NEEDED] и [VERIFY] с предложениями |

### Форматирование и экспорт

| Скилл | Что делает |
|-------|-----------|
| `/export-thesis [slug] [name]` | Собирает все главы в один DOCX с оглавлением и форматированием по FORMAT.md |
| `/create-reference-doc <проект>` | FORMAT.md → reference.docx для Pandoc (только шаблон) |

### Синхронизация с репозиторием

| Скилл | Что делает |
|-------|-----------|
| `/sync` | Закоммитить все изменения и запушить в remote |
| `/sync pull` | Получить изменения из remote |
| `/sync checkpoint` | Быстрый локальный коммит без пуша — сохранить прогресс сессии |

> Рекомендуется: `/sync checkpoint` каждые 30-45 минут в длинных сессиях, `/sync` в конце.

## Структура подпроекта

```
projects/<slug>/
├── CONTEXT.md          ← параметры работы — заполнить первым
├── FORMAT.md           ← форматирование — проверить/настроить
├── AUTHOR.md           ← голос автора — вставить фрагменты текстов
├── STYLE.md            ← правила письма — список запретов + примеры
├── reference.docx      ← генерируется автоматически из FORMAT.md
│
├── outline/
│   └── outline_v1.md   ← структура работы (генерируется /build-outline)
│
├── chapters/           ← готовые и черновые разделы
│   └── 01_vvedenie.md  ← именование: NN_slug.md
│
├── sources/
│   ├── files/          ← сюда кидаешь PDF и DOCX с литературой
│   ├── notes/          ← карточки источников (создаются /process-source)
│   │   └── _template.md
│   ├── bibliography.md ← читаемый список источников
│   └── bibliography.bib← BibTeX для Pandoc (пополняется через /process-source)
│
├── drafts/             ← черновики, эксперименты, промежуточные версии
├── review/             ← результаты ревью (создаются /review-section)
└── output/             ← готовые DOCX после экспорта
```

## Флаги в тексте

Ставятся Claude при генерации, закрываются вручную или через `/check-citations`:

| Флаг | Значение |
|------|---------|
| `[VERIFY]` | Факт, дата, цифра — требует проверки по первоисточнику |
| `[SOURCE NEEDED]` | Здесь нужна ссылка или цитата |
| `[WEAK ARG]` | Аргумент слабый, требует усиления или убирается |
| `[UNCLEAR]` | Формулировка неточная или двусмысленная |

Перед сдачей работы — ни одного незакрытого флага.

## Типичная последовательность для новой работы

```
1. /new-project              → создать папку
2. Заполнить CONTEXT.md      → тема, ИВ, дедлайн, методология
3. Заполнить FORMAT.md       → шрифт, поля (если отличается от ГОСТ)
4. Заполнить AUTHOR.md       → вставить фрагменты своих текстов
5. Заполнить STYLE.md        → запреты + образцы
6. /research-plan            → план этапов с дедлайнами
7. /find-sources <тема>      → направления для поиска литературы
8. Скопировать PDF в sources/files/
9. /process-source <файл>    → для каждого источника
10. /build-outline           → структура работы (3 варианта → выбор)
11. /write-section <раздел>  → черновики глав
12. /find-quote / /check-citations → закрыть [SOURCE NEEDED]
13. /review-section <файл>   → ревью каждой главы
14. /literature-review       → обзор литературы (если отдельная глава)
15. /consistency-check       → проверка связности всей работы
16. /style-check <файл>      → финальная вычитка стиля
17. /abstract-write          → аннотация
18. /export-thesis           → собрать все главы в DOCX
```
