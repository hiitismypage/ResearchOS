Аргумент: тема обзора, номер раздела или "весь обзор". Например:
- `/literature-review теории репрезентации`
- `/literature-review 1.1`
- `/literature-review весь обзор`

## Перед написанием

Обязательно прочитай:
1. `projects/<проект>/CONTEXT.md` — тема, исследовательский вопрос, методология
2. `projects/<проект>/STYLE.md` — стиль и язык
3. `projects/<проект>/outline/outline_v1.md` — место обзора в структуре
4. Все файлы из `projects/<проект>/sources/notes/` (кроме `_template.md`)

Если `notes/` пустой или содержит менее 3 обработанных источников — предупреди и
предложи сначала запустить `/process-source` для загруженных PDF.

## Логика построения обзора

### Выбор принципа группировки

Выбери что лучше для этой работы и объясни выбор:

**Тематические блоки** — когда есть несколько дискуссий / школ мысли (рекомендуется для большинства случаев)

**Хронологический порядок** — когда важна эволюция взглядов на проблему

**Смешанный** — тематические блоки внутри хронологических периодов

### Структура каждого блока

```
[Тема блока]
→ Что известно / на чём сходятся авторы
→ Где расходятся / дискуссия
→ Что остаётся неизученным (gap)
```

### Требования к тексту

- Каждый абзац — одна группа источников или один теоретический спор
- Показывать согласие/противоречия между авторами (не просто "Иванов пишет... Петров пишет...")
- Ввод цитат по STYLE.md
- `[SOURCE NEEDED]` там где нужна точная цитата из оригинала (не из карточки)
- Завершить лакуной: что не изучено, что делает твою работу необходимой
