Ты пишешь черновик конкретного раздела исследовательской работы.

## Аргумент скилла

Название раздела или номер из outline. Например:
- `/write-section Введение`
- `/write-section 2.1 Теоретические основы`

## Что прочитать перед написанием (обязательно)

1. `projects/<проект>/CONTEXT.md` — тема, исследовательский вопрос, методология
2. `projects/<проект>/AUTHOR.md` — голос автора, типичные конструкции
3. `projects/<проект>/STYLE.md` — правила письма, запрещённые обороты
4. `projects/<проект>/outline/outline_v1.md` — место раздела в общей структуре
5. Уже написанные главы (если есть) — для согласованности и отсутствия повторов

Если какой-то из этих файлов не заполнен — сообщи об этом до начала написания.

## Правила написания

### Структура академического раздела

Каждый содержательный абзац должен содержать:
- **Тезис** (первое предложение — главная мысль абзаца)
- **Развёртывание** (аргумент, данные, логика)
- **Закрытие** (вывод абзаца или переход к следующему)

### Флаги

Используй флаги прямо в тексте, не убирай их:
- `[VERIFY]` — факт, дата, цифра или утверждение, которое нужно проверить
- `[SOURCE NEEDED]` — место где нужна цитата или ссылка
- `[WEAK ARG]` — аргумент требует усиления
- `[UNCLEAR]` — формулировка неточная, требует доработки

### Чего избегать

- Не выдумывать источники — только флаги [SOURCE NEEDED]
- Не утверждать факты неуверенно без флага [VERIFY]
- Не отступать от STYLE.md и AUTHOR.md
- Не повторять содержание других разделов
- Не писать "в данном разделе будет рассмотрено" — просто рассматривай

## После написания — self-review

После черновика добавь блок:

```
---
## Self-review

**Сильные стороны этого черновика:**
- [что получилось хорошо]

**Слабые места / требуют доработки:**
- [что нужно усилить]

**Флаги:**
- [VERIFY]: [список фактов для проверки]
- [SOURCE NEEDED]: [список мест где нужны источники]
```

## Сохранение

Сохрани в `projects/<проект>/chapters/<NN>_<slug-раздела>.md`

Например: `chapters/01_vvedenie.md`, `chapters/02_1_teoriya.md`

Если файл уже существует — спроси: перезаписать или создать новую версию.
