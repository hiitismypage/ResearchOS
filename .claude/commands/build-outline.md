Ты помогаешь построить структуру (оглавление) исследовательской работы.

## Перед началом

Обязательно прочитай:
- `projects/<активный_проект>/CONTEXT.md` — тема, тип, методология, требования
- Существующий outline (если есть) — `projects/<активный_проект>/outline/`

Если CONTEXT.md не заполнен (особенно поля "Исследовательский вопрос" и "Методология") —
скажи об этом и попроси заполнить перед продолжением.

## Что создать

### Три варианта структуры

Для каждого варианта:
1. Полное оглавление с подразделами (2 уровня)
2. Примерный объём каждой части (в % от общего или страницах)
3. **Обоснование логики:** как этот план отвечает на исследовательский вопрос
4. **Плюсы этого варианта**
5. **Минусы / риски**

### Принципы хорошей структуры

- Каждая глава — логический шаг к ответу на исследовательский вопрос
- Порядок разделов обоснован (не произвольный)
- Объём соответствует требованиям из CONTEXT.md
- Нет дублирования между главами
- Введение ставит вопрос, заключение отвечает на него
- Обзор литературы встроен в логику, а не "пристёгнут" отдельно

### Типичные структуры по типу работы

**Дипломная/магистерская:**
Введение → Теоретическая глава → Методологическая глава → Эмпирическая глава → Заключение

**Статья:**
Abstract → Введение → Обзор литературы → Методы → Результаты → Обсуждение → Заключение

**Эссе:**
Вводный тезис → Аргумент 1 → Аргумент 2 → Аргумент 3 → Контраргумент + опровержение → Заключение

**Курсовая:**
Введение → Теоретическая часть → Практическая часть → Заключение

## Итерация

После показа трёх вариантов:
1. Спроси что понравилось в каждом
2. Предложи создать гибридный вариант если нужно
3. Итерируй до финального плана

## Сохранение

Когда пользователь утвердит финальный outline — сохрани его в:
`projects/<slug>/outline/outline_v1.md`

Формат файла:
```markdown
# Структура работы — <название>
_Версия 1, дата_

## Оглавление

1. Введение (~ X стр.)
   1.1. ...
   1.2. ...

2. ...

## Обоснование структуры

[Краткое объяснение логики плана]

## Следующий шаг

Используй /write-section для работы с конкретными разделами.
```
