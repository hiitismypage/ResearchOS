Анализирует целостность работы: связаны ли главы между собой и с исследовательским вопросом.

## Перед началом

Прочитай:
1. `projects/<проект>/CONTEXT.md` — исследовательский вопрос, цель, задачи
2. `projects/<проект>/outline/outline_v1.md` — задуманная структура
3. Для каждой главы в `chapters/`: **только заголовки разделов + первый абзац + последний абзац**

ВАЖНО для токенов: не читать главы целиком — только заголовки и граничные абзацы.
Если нужен полный анализ конкретной главы — пользователь запустит `/review-section`.

## Что проверяется

### 1. Структурная карта работы

Строй карту:
```
ИССЛЕДОВАТЕЛЬСКИЙ ВОПРОС: "..."

Глава 1: [название]
  Задача главы (из outline): ...
  Реальное содержание (из первого абзаца): ...
  Вывод главы (из последнего абзаца): ...
  Статус: ✅ соответствует outline / ⚠️ расхождение / ❓ непонятно

Глава 2: ...
...

→ Заключение: [как связывается с ИВ]
```

### 2. Чеклист связности

- [ ] Каждая глава отвечает на часть исследовательского вопроса?
- [ ] Выводы глав логически ведут к общему выводу?
- [ ] Нет повторов одного и того же в разных главах?
- [ ] Введение и заключение согласованы (одни и те же задачи)?
- [ ] Все задачи из введения закрыты в тексте работы?
- [ ] Нет "висящих" разделов без очевидной функции?

### 3. Проблемы и рекомендации

```
## Выявленные проблемы

### Критические
1. [Проблема]: ...
   Рекомендация: ...

### Значимые
2. ...

### Мелкие
3. ...

## Следующие шаги
[конкретный список что нужно сделать]
```

Если работа ещё не написана полностью — анализируй то что есть и отмечай
какие главы отсутствуют.
