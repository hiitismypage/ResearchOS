Ты помогаешь создать новый исследовательский подпроект в системе ResearchOS.

## Шаги

1. **Запроси у пользователя:**
   - Рабочее название работы (для slug папки — латиницей или транслитерацией, без пробелов)
   - Полное название работы
   - Тип работы: дипломная / магистерская / курсовая / статья / эссе / другое

2. **Создай структуру папок:**
   ```
   projects/<slug>/
   ├── CONTEXT.md
   ├── AUTHOR.md
   ├── STYLE.md
   ├── outline/
   ├── chapters/
   ├── sources/
   │   ├── bibliography.md
   │   └── notes/
   ├── drafts/
   └── review/
   ```

3. **Скопируй шаблоны** из `templates/` в созданную папку:
   - `templates/project_context.md` → `projects/<slug>/CONTEXT.md`
   - `templates/author_profile.md` → `projects/<slug>/AUTHOR.md`
   - `templates/style_guide.md` → `projects/<slug>/STYLE.md`

4. **Заполни автоматически** в CONTEXT.md:
   - Название работы (из ввода пользователя)
   - Тип работы (из ввода пользователя)
   - Дату создания

5. **Создай пустой bibliography.md** с заголовком и структурой:
   ```markdown
   # Список источников — <название работы>

   ## Основные источники

   ## Дополнительные источники

   ## Онлайн-ресурсы
   ```

6. **Создай .gitkeep** в пустых папках (outline, chapters, sources/notes, drafts, review).

7. **Выведи чеклист** что нужно заполнить вручную:

   ```
   ✅ Создана папка projects/<slug>/

   📝 НУЖНО ЗАПОЛНИТЬ ВРУЧНУЮ:

   1. CONTEXT.md — САМОЕ ВАЖНОЕ:
      □ Исследовательский вопрос / тема
      □ Гипотеза
      □ Методология
      □ Дедлайн сдачи
      □ Требования к объёму
      □ Требования к оформлению (ГОСТ/APA/другое)
      □ Научный руководитель
      □ Что уже есть (материалы, черновики)
      □ Ограничения от руководителя

   2. AUTHOR.md:
      □ Стиль этого автора (можно взять из Me.md и адаптировать)
      □ Примеры текстов (вставить 2-3 абзаца для ориентира)

   3. STYLE.md:
      □ Тон и уровень формальности
      □ Запрещённые обороты
      □ Формат ссылок

   🚀 СЛЕДУЮЩИЙ ШАГ: когда заполнишь CONTEXT.md — запусти /build-outline
   ```

Используй инструменты Write, Edit, Bash(mkdir) для создания файлов и папок.
Читай файлы из templates/ с помощью Read перед копированием.
